# Plan: The Energy Dome raid (fireballs from the sky vs a community shield)

Locked contract. All lanes build against THIS. Do not change shared shapes without
editing this file first.

## The idea (from the player)

The city sits under a translucent protective **dome** (an ozone-layer energy shield).
A raid is **5-6 fireballs falling from the top**. Each fireball has a rolled **power**;
each dome **segment** has a current **shield**. Per fireball:

- `power <= segment shield` -> **BLOCKED**: the fireball crashes on the dome, the
  segment loses integrity (ripple, no city damage).
- `power > segment shield`  -> **PENETRATES**: it punches through and destroys a
  house (feeds the existing reconstruction queue) + costs souls/resources.

The dome's shield comes from **daily challenges** (community charge) and is repaired
by an **earned shield pool** (per accepted contribution) that auto-mends the
**most-damaged segment** when it crosses a threshold. Ownership of houses is never
lost (reconstruction system, unchanged).

## Model

- Dome = **6 segments**, each `shield in [0, 100]`. Fresh city: all at `segmentStart`.
- **Charge (daily challenges):** each completed daily challenge adds `chargePerChallenge`
  to ONE seeded segment (atomic `hIncrBy`; clamped to `segmentMax` on read). Reuses the
  existing once-per-user/day challenge-done claim in `/init`.
- **Repair pool (per accepted contribution):** a `shield` accumulator, `+shieldPerContribution`
  per accepted contribution (atomic `hIncrBy`). When `shield >= repairThreshold` AND a
  damaged segment (`< segmentMax`) exists, auto-repair: fully restore the MOST-damaged
  segment and `shield -= repairThreshold` (optimistic CAS; may repeat while affordable).
- **Raid volley (deterministic):** seed = `(worldSeed ^ imul(cycle,40503) ^ imul(day,97)) >>> 0`
  (same seed reconstruction already uses). Count = `min + (seed % (max-min+1))`. For each
  fireball i: `segment = h(seed,i,'seg') % 6`, `power = powerMin + h(seed,i,'pow') % (powerMax-powerMin+1)`.
  `blocked = power <= dome[segment]`. Blocked -> `dome[segment] = max(0, dome[segment]-blockDrain)`.
  Else `penetrations++`.
- **Losses** (softened by the physical layer `dampen = guard_wall_actions*3 + wall raidDampen`,
  exactly as today): for each stat, `loss = max(0, penetrations * perPenetration[stat] - dampen)`.
  Applied to the post-clamp city; `threat = postRaidThreat`.
- **Outcome:** `penetrations === 0` -> `held`; `> 0` and city survives -> `breach`;
  city falls (`population <= fall.populationThreshold`) -> `fallen`. (Same enum the
  reconstruction selection already consumes.)
- Dome (segment shields) is drained ONLY by the raid (at midnight resolve). Charge and
  repair happen outside resolve (in `/init` and `/action`). No global energy separate
  from segment shields; HUD "Dome Energy" = average segment % .

```ts
export type Fireball = { power: number; segment: number; blocked: boolean };
export type DomeVolley = { fireballs: Fireball[]; segmentsAfter: number[]; penetrations: number };

// clamp a raw segment array (from store, possibly over/under) to [0, segmentMax], length 6
export const normalizeSegments = (raw: (number|undefined)[]): number[];
export const freshSegments = (): number[];                    // 6 x segmentStart
export const energyPct = (segments: number[]): number;         // 0..100 average, rounded
export const mostDamagedSegment = (segments: number[]): number | null; // lowest < segmentMax, else null
export const chargeSegmentIndex = (userId: string, day: number, worldSeed: number): number; // 0..5
export const resolveVolley = (seed: number, segments: number[]): DomeVolley; // pure, does not mutate input
// auto-repair: given pool + segments, return how many repairs and the resulting arrays
export const applyRepairs = (segments: number[], pool: number):
  { segments: number[]; pool: number; repaired: number[] }; // repaired = segment indices restored, in order
```

## Client scene (src/client/scene.ts) LANE A (agent)

Evolve the raid from "arc over the wall" to the DOME model. Keep it phone-safe (pooled,
no per-frame allocation, <=2 transient lights, no postprocessing). New/updated handle:

- `setDome(segments: number[]): void` — a translucent hemisphere over the city, split into
  6 arc panels. Each panel's opacity/crack reflects its shield (100 = clear bright, low =
  dim + hairline cracks, 0 = shattered gap). Idempotent; safe pre-scene.
- `playRaidCinematic(opts: { outcome; fireballs: RaidFireball[]; hitHouseIndices: number[] }): void`
  — replace the arc trajectories with fireballs **falling from high above straight/steeply down**
  toward the dome. For each fireball, aim at its `segment`. If `blocked`: it strikes the dome
  panel with an energy RIPPLE + spark and stops (panel flares, then dims). If not blocked: it
  PIERCES the panel (panel flashes white then gaps) and continues down to a house in
  `hitHouseIndices`, landing with the existing impact/ember/smoke + shake. `outcome` still tints
  sky red-orange; 'fallen' hazes the town. Reuse the existing fireball/impact/plume/smoke pools.
- `repairDomeSegment(segment: number): void` — heal one panel back to full over ~1s (rising
  shimmer), clearing its cracks. Self-terminating.
- Keep `setHouseDamage` / `rebuildHouse` (reconstruction) exactly as-is.

Update the `VillageHandle` type + the returned object. Never throw from any of these.
